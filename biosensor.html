Research Overview

During this research term, I focused on the technical implementation of a real-time biosensing platform. Our work bridged the gap between low-level firmware development and high-level mobile user interfaces, creating a cohesive system for monitoring cardiovascular biomarkers.

The project involved developing a microcontroller-based system to collect, log, and display electrochemical and temperature data in real-time.

Embedded Firmware & Hardware Communication

I served a researcher for firmware integration on the STM32WB55RG microcontroller platform. Using STM32CubeIDE, I helped configure system clocks, GPIOs, and peripherals to manage complex data flows.

  • SPI Integration: Modified and adapted SDKs from Analog Devices to enable communication with the AD5940 potentiostat for precise electrochemical signal measurement.
  • I²C & DMA: Implemented I²C protocols for temperature sensing, utilizing Direct Memory Access (DMA) to ensure non-blocking, efficient data transfers essential for real-time systems.
  • Data Persistence: Integrated a FATFS file system for microSD card logging, developing a unique timestamp-based naming convention to prevent data overwriting during long-term monitoring sessions.

Mobile Application Development

To make the data accessible to users, I contributed to a cross-platform mobile application built with Flutter.

  • Designed front-end UI components using Dart to improve navigability and data readability.
  • Developed asynchronous data handling to process real-time streams coming from the microcontroller via Bluetooth Low Energy (BLE).
  • Built custom interface sections, such as a sampling interval configuration page, to give users control over sensor data frequency.
